# FlipCard

> The first primitive in an adaptive, manifest schema-driven, composable UI system.

[![License: MIT](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![CI](https://github.com/microsoft/flipcard/actions/workflows/ci.yml/badge.svg)](https://github.com/microsoft/flipcard/actions/workflows/ci.yml)
[![Pages](https://github.com/microsoft/flipcard/actions/workflows/deploy.yml/badge.svg)](https://github.com/microsoft/flipcard/actions/workflows/deploy.yml)

FlipCard decouples **design ⟂ schema ⟂ workflow**. The front face renders the design; the back face exposes the manifest schema; a click flips between them. Every card is described by a portable JSON manifest validated against a public schema, rendered by your framework of choice (React, vanilla web component, server-side), themed with shared design tokens, and authorable by humans or LLMs through an MCP server.

- **Documentation site:** <https://microsoft.github.io/flipcard/>
- **Live showcase:** <https://microsoft.github.io/flipcard/showcase/>
- **Storybook:** <https://microsoft.github.io/flipcard/storybook/>
- **Manifest schema:** <https://microsoft.github.io/flipcard/schema/v0.1.json>

---

## Why FlipCard

UI systems usually entangle three concerns: how a component looks, what data shape it expects, and what it does. FlipCard separates them:

| Layer | What it owns | Where it lives |
|---|---|---|
| **Design** | Visual rendering of the front face | Renderer (React, vanilla, future) + theme tokens |
| **Schema** | The manifest payload describing the card | `@microsoft/flipcard-core` JSON Schema |
| **Workflow** | Events and actions a card binds to | `workflow` block of the manifest |

The same manifest renders identically across React and vanilla, can be generated by an LLM through the MCP server, and is statically validatable.

## Packages

This is a TypeScript monorepo (npm workspaces).

| Package | Path | Purpose |
|---|---|---|
| [`@microsoft/flipcard-core`](packages/core) | `packages/core` | Framework-agnostic types, controller state machine, JSON Schema, and runtime validator. |
| [`@microsoft/flipcard-themes`](packages/themes) | `packages/themes` | Shared design tokens (light, dark, midnight-sapphire) consumed by every renderer. |
| [`@microsoft/flipcard-react`](packages/react) | `packages/react` | React components: `<FlipCard>` and `<FlipCardCatalog>`. |
| [`@microsoft/flipcard`](packages/vanilla) | `packages/vanilla` | Framework-neutral `<flip-card>` web component plus an asset library and renderer helpers. |
| [`@microsoft/flipcard-mcp`](packages/mcp-server) | `packages/mcp-server` | Model Context Protocol server that lets agents list categories, fetch the schema, validate manifests, and generate starter cards. |
| Showcase app | `apps/showcase` | Vite + React app that consumes every package — also the GitHub Pages homepage. |

## Quick start

### Install

```sh
# pick what you need
npm install @microsoft/flipcard-core
npm install @microsoft/flipcard-react
npm install @microsoft/flipcard           # vanilla web component
npm install @microsoft/flipcard-themes
```

### React

```tsx
import { FlipCard } from '@microsoft/flipcard-react';
import '@microsoft/flipcard-themes/styles/light.css';

const asset = {
  id: 'welcome',
  category: 'teal',
  theme: 'light',
  design: { title: 'Hello FlipCard', summary: 'Click to see the manifest.' },
  manifest: {
    $schema: 'https://microsoft.github.io/flipcard/schema/v0.1.json',
    id: 'welcome',
    title: 'Welcome',
  },
};

export function App() {
  return <FlipCard asset={asset} interactive onFlip={(s) => console.log(s)} />;
}
```

### Vanilla / web component

```html
<script type="module">
  import { defineFlipCard } from '@microsoft/flipcard';
  defineFlipCard();
</script>

<flip-card front-title="Hello" back-title="Manifest" show-flip-hint></flip-card>
```

### Core controller (no UI)

```ts
import { FlipCardController, validateManifest } from '@microsoft/flipcard-core';

const controller = new FlipCardController({ defaultState: 'front' });
controller.subscribe((state) => console.log('flipped to', state));
controller.flip(); // 'back'

validateManifest({ id: 'demo', title: 'Hello' }); // { valid: true, errors: [] }
```

### MCP server (for agents)

```sh
npx @microsoft/flipcard-mcp
```

Tools exposed: `list_categories`, `get_schema`, `validate_manifest`, `generate_manifest`, `describe_component`.

See [Docs → MCP server](https://microsoft.github.io/flipcard/packages/mcp-server) for full client-configuration examples.

## Development

Requirements: Node ≥ 20, npm ≥ 10.

```sh
git clone https://github.com/microsoft/flipcard.git
cd flipcard
npm install

# build every workspace
npm run build

# run the showcase locally
npm run dev

# Storybook
npm run storybook

# tests
npm test            # run once
npm run test:watch  # watch mode

# lint, format, typecheck
npm run lint
npm run format
npm run typecheck

# documentation site
cd docs && npm install && npm start
```

### Useful scripts

| Script | What it does |
|---|---|
| `npm run build` | Build all libraries, the showcase, and the MCP server. |
| `npm run build:libs` | Build only the publishable libraries. |
| `npm run build:showcase` | Build the Vite showcase app. |
| `npm run build:storybook` | Build the static Storybook bundle. |
| `npm run dev` | Run the showcase in dev mode. |
| `npm run storybook` | Run Storybook on port 6006. |
| `npm test` | Run the Vitest suite once. |
| `npm run lint` / `lint:fix` | ESLint across the monorepo. |
| `npm run format` / `format:check` | Prettier across the monorepo. |
| `npm run typecheck` | TypeScript no-emit across every workspace. |
| `npm run clean` | Remove `dist/` outputs and Storybook build. |
