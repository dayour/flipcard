# @microsoft/flipcard-desktop

**FlipCard Studio** вЂ” a native Windows desktop wrapper around the FlipCard
showcase web app, built with [Tauri 2.x](https://tauri.app/).

It bundles the Vite/React showcase from `apps/showcase` so designers and
developers can browse the FlipCard library offline, and is the host process for
future tooling (notably the bundled `@microsoft/flipcard-mcp` server, which
will be spawned as a Tauri sidecar so the desktop app can serve agents).

## Prerequisites

- Node 20+ and npm (the workspace is npm-based)
- [Rust](https://www.rust-lang.org/tools/install) via `rustup` (stable)
- Visual Studio 2022 **Build Tools** with the *Desktop development with C++*
  workload (required by the MSVC linker Tauri uses on Windows)
- [WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/)
  (the bundle config uses `embedBootstrapper`, so installers fetch it on demand)
- For MSIX: Windows 10/11 SDK (provides `MakeAppx.exe` and `SignTool.exe`) вЂ”
  see [`MSIX.md`](./MSIX.md)

## Develop

From `apps/desktop/`:

```bash
npm run dev
```

Tauri runs `beforeDevCommand` (which starts the showcase Vite dev server on
`http://localhost:5173`) and then opens the native window pointed at it. Edits
to the showcase hot-reload as usual.

## Build

```bash
npm run build
```

Produces installers under
`apps/desktop/src-tauri/target/release/bundle/`:

- `msi/FlipCard Studio_0.1.0_x64_en-US.msi` вЂ” Wix-based MSI
- `nsis/FlipCard Studio_0.1.0_x64-setup.exe` вЂ” NSIS installer

## MSIX

```pwsh
pwsh apps/desktop/msix/build-msix.ps1
```

See [`MSIX.md`](./MSIX.md) for prerequisites, signing, install, and Store
submission notes.

## Icons

Placeholder icons live in `src-tauri/icons/` and are generated by:

```bash
npm run gen:icons
```

This produces a teal-on-white set sized for Tauri (`32x32.png`, `128x128.png`,
`128x128@2x.png`, `icon.png`, `icon.ico`, plus a stub `icon.icns`). **Replace
them with branded artwork before shipping** вЂ” the easiest path is to drop a
1024Г—1024 source PNG and run `npx @tauri-apps/cli icon ./flipcard-source.png`.

## Future work

- **MCP sidecar.** Bundle the `@microsoft/flipcard-mcp` server as a Tauri
  sidecar (`bundle.externalBin`) so launching FlipCard Studio also exposes the
  catalog over MCP for local agents (Copilot, Claude Desktop, etc.). The
  Rust side will spawn it via `tauri-plugin-shell` and forward stdio.
- **Auto-update.** Add `tauri-plugin-updater` once a release endpoint exists.
- **Real branding.** Replace placeholder icons and refresh `AppxManifest.xml`
  visual assets.
- **Code signing.** Wire `WINDOWS_CERTIFICATE` secret into the release
  workflow.

## CI
